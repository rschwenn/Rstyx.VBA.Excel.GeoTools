Folgende Berechnungen werden unterstützt:

Fehler und Verbesserungen
·    Vorhandene Fehlerspalten werden als Verbesserungen eintragen (unter Umkehrung des Vorzeichens).
·    Vorhandene Verbesserungsspalten werden als Fehler eintragen (unter Umkehrung des Vorzeichens).
·    Aus Soll- und Istwerten werden Verbesserungen und Fehler berechnet.


Trassenkoordinaten <==> Gleissystem
Die entsprechende Transformation wird für alle Wertstati durchgeführt: Soll, Ist, Fehler, Verbesserung.
=> Es wird immer die Soll-Überhöhung verwendet!
=> Das Vorzeichen des Radius entscheidet darüber, welche Schiene überhöht ist. ("-" entspricht Linksbogen, also rechte Schiene überhöht.)
=> Ist das Projektdatenfeld "Basis für Überhöhung [m]" in der Tabelle verfügbar, so findet es Anwendung! Anderenfalls wird der konfigurierte Standardwert verwendet.



InfoText auswerten
Als InfoText wird der Inhalt der Spalte „Punkt Erläuterung Lage“ (Pkt.Erl.L) angesehen.
1.           Enthält der InfoText eine iTrassen-Codierung, werden die verfügbaren Informationen extrahiert (Vermarkung für iGeo-Absteckbuch, Punktart, Vermarkung, Überhöhung, Spurweite). Ist die Option „Überschreiben“ aktiv, wird die iTrassen-Codierung aus dem InfoText entfernt.
2.           Ist die Punktart dann noch unbekannt, wird versucht, sie anhand bestimmter Angaben in der Spalte „Punktart Bezeichnung kurz“ (Pkt.Art.Bez1) zu erkennen.
3.           Ist die Punktart dann noch unbekannt, wird der InfoText auf allgemeine Punktart- und Überhöhungsangaben („u=xx“) untersucht. Eine gefundene Überhöhungsangabe wird aus dem InfoText entfernt, sofern die Option „Überschreiben“ aktiv ist.

Eine erkannte Punktart wird in Spalte „Punktart Bezeichnung lang“ (Pkt.Art.Bez2) eingetragen. Die Kurzbezeichnung wird in Spalte „Punktart Bezeichnung kurz“ eingetragen, falls diese leer ist.
